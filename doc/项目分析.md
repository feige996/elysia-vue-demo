# 项目分析报告

## 项目概述

这是一个名为 `vue3-bun-elysia-demo` 的全栈开发模板，采用 **Vue3 + Elysia + Bun + AlovaJS** 技术栈。这是一个 **Monorepo** 结构，使用 `bun workspaces` 管理前后端项目。

## 目录结构

```
elysia-vue-demo/
├── api/                    # 后端项目 (Elysia + Bun)
│   ├── src/
│   │   ├── app/            # 应用核心 (中间件、插件)
│   │   ├── modules/        # 业务模块 (user、article)
│   │   ├── infra/          # 基础设施 (数据库)
│   │   ├── shared/         # 共享代码 (auth、logger、types)
│   │   └── index.ts
│   ├── tests/              # 测试 (unit + integration)
│   └── package.json
├── web/                    # 前端项目 (Vue3 + Vite)
│   ├── src/
│   │   ├── api/           # API 请求封装 (alova)
│   │   ├── views/         # 页面组件
│   │   ├── App.vue
│   │   └── main.ts
│   └── package.json
├── doc/                    # 技术文档
├── package.json            # 根目录 workspace 配置
└── README.md
```

## 架构设计

### 后端分层架构

后端采用分层架构：

- **Controller** ([user.controller.ts](file:///Users/burtlai/Desktop/elysia-vue-demo/api/src/modules/user/user.controller.ts), [article.controller.ts](file:///Users/burtlai/Desktop/elysia-vue-demo/api/src/modules/article/article.controller.ts)) - 处理请求/响应
- **Service** ([user.service.ts](file:///Users/burtlai/Desktop/elysia-vue-demo/api/src/modules/user/user.service.ts)) - 业务逻辑
- **Repository** ([user.repository.ts](file:///Users/burtlai/Desktop/elysia-vue-demo/api/src/modules/user/user.repository.ts), [article.repository.ts](file:///Users/burtlai/Desktop/elysia-vue-demo/api/src/modules/article/article.repository.ts)) - 数据访问

### 中间件栈

中间件按以下顺序加载 ([app/index.ts](file:///Users/burtlai/Desktop/elysia-vue-demo/api/src/app/index.ts#L14-L18))：

```
cors → di → logger → error → auth → routes
```

## 核心特性

1. **依赖注入** - 使用 `elysia-di` 实现，控制反转
2. **JWT 认证** - [token-auth.ts](file:///Users/burtlai/Desktop/elysia-vue-demo/api/src/shared/auth/token-auth.ts)
3. **请求验证** - 使用 Zod 进行 DTO 校验
4. **日志系统** - Pino + 按天落盘
5. **数据库迁移** - Drizzle Kit
6. **测试** - Bun test (unit + integration)
7. **CI/CD** - GitHub Actions 配置

## 常用命令

```bash
bun install          # 安装依赖
bun run dev          # 并行启动前后端
bun run build        # 构建前后端
bun run typecheck    # 类型检查
bun run test         # 运行测试
```

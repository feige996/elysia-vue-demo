1. 能提升页面开发效率与一致性的改造。

## 对比结论（聚焦可迁移点）

- `gi-demo` 的优势不在后端能力，而在前端“模板化开发体验”（页面骨架 + hooks + 自动映射）。
- Harbor 已具备更强的后端与治理基础，短板主要在“前端开发效率的一致性抽象”。
- 建议路线：**借鉴开发模式，不迁移技术栈**（不引入 Arco 组件库，不替换现有 Naive UI）。

## P0（本月可落地）文件级清单

## P0-1 路由组件自动映射（减少手写 `componentKey -> import` 分支）

### 背景

当前 `admin/src/router/index.ts` 中 `resolveMenuComponent` 采用多分支硬编码映射。随着页面增长，维护成本会线性上升。

### 改造文件

- `admin/src/router/index.ts`（改）
- `admin/src/router/view-map.ts`（新）
- `admin/src/router/__tests__/view-map.test.ts`（新，可选，若当前 admin 未接测试则后置到 api 侧契约测试）

### 实施要点

- 使用 `import.meta.glob('../views/**/*.vue')` 构建视图映射表。
- 菜单 `component` 继续沿用后端给出的 key（如 `system/user/index`）。
- 保留兜底页（未命中映射时跳默认页并记录告警日志）。

### 工作量与风险

- 预计工作量：`0.5 ~ 1 人日`
- 风险等级：`中`
- 风险点：
  - 历史 `componentKey` 命名不规范导致映射失败。
- 缓解：
  - 增加映射失败统计日志；
  - 首次发布保留旧分支兜底一版。

### 验收标准

- 登录后动态路由可正常注入。
- 用户、角色、菜单、部门、字典、监控等既有页面路由全量可打开。
- 无权限时仍按现有策略跳转 `403`。

## P0-2 列表页查询/重置/加载交互统一（抽象 `useListQuery`）

### 背景

多个列表页存在重复代码：`loading/error/query/reset/fetch`，维护成本较高，行为细节不一致。

### 改造文件

- `admin/src/composables/useListQuery.ts`（新）
- `admin/src/views/UserManagement.vue`（改）
- `admin/src/views/RoleManagement.vue`（改）
- `admin/src/views/ApiCatalogManagement.vue`（改）
- `admin/src/views/AuditLogManagement.vue`（改）

### 实施要点

- 抽象统一状态：
  - `query`
  - `loading`
  - `errorText`
  - `search()`
  - `reset()`
- 保持页面 API 入参与返回结构不变，仅统一状态管理方式。

### 工作量与风险

- 预计工作量：`1.5 ~ 2.5 人日`
- 风险等级：`低`
- 风险点：
  - 个别页面存在特殊查询逻辑（如联动筛选）被抽象过度。
- 缓解：
  - composable 通过泛型 + 回调保留扩展位，不强制一刀切。

### 验收标准

- 改造页面查询/重置行为一致。
- 查询条件变更后分页重置规则一致（若有分页）。
- TypeScript 类型检查无回退。

## P0-3 CRUD 页面脚手架模板（减少新页起步成本）

### 背景

你们已有 `DataTablePage` / `FormDrawer` / `SearchBar` 基础组件，建议新增“页面模板约定”，对齐 `gi-demo` 的高复用页面模式。

### 改造文件

- `admin/src/views/_templates/CrudPageTemplate.vue`（新）
- `admin/src/components/crud/README.md`（新）
- `doc/README.md`（改，补一条“新页面推荐模板”入口）

### 实施要点

- 提供可复制模板：
  - 搜索区
  - 工具栏
  - 表格区
  - 抽屉表单
  - 权限按钮示例（`v-permission`）
- 新增页面默认走模板，旧页面渐进迁移。

### 工作量与风险

- 预计工作量：`0.5 ~ 1 人日`
- 风险等级：`低`

### 验收标准

- 新增一个示例页可在 `0.5 人日` 内完成“可查询 + 可新增”最小链路。
- 团队成员按模板开发时不需要重复搭页面骨架。

### 当前验收结论（2026-04）

- 模板与配套说明已落地，可作为新页面默认起手骨架。
- 已覆盖查询、分页、重置、抽屉表单、权限按钮等高频场景。
- 后续按业务页落地时，只需替换 API 与字段，不需重复搭建页面结构。

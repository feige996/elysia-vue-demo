# 两周落地改造清单

目标：先稳住可运行与可维护，再逐步提升文档与工程质量。  
周期：10 个工作日（2 周）。  
优先级：`P0` 必须完成，`P1` 建议完成，`P2` 有余力再做。

## P0（第 1 周，必须完成）

### 1) 配置与环境收口（Day 1）

- 统一端口与 API 基地址读取规则
- 校验 `.env/.env.example/docker-compose/CI` 一致性

### 2) 登录链路稳定性加固（Day 1-2）

- 守卫中的初始化与动态路由幂等化
- `bootstrapAuthContext` 增加 in-flight 锁

### 3) 请求层防循环刷新（Day 2-3）

- `authRequest` 仅在明确 401 时刷新
- 刷新接口调用不走 `authRequest`
- 最大重试次数限制为 1

### 4) 菜单树构建防环保护（Day 3）

- `toMenuTree` 增加 visited 检测
- 发现环时降级处理并记录日志

### 5) 回归测试补齐（Day 4-5）

- admin 最小 e2e：登录 -> 拉权限/菜单 -> 进入首页
- request client 单测：401 刷新成功/失败
- API 集成覆盖 `/permissions/current`、`/menus/tree`

## P1（第 2 周，建议完成）

### 6) OpenAPI 文档质量提升（Day 6-7）

- 路由统一补齐 `detail.tags/summary/description/security`

### 7) OpenAPI 漂移校验强化（Day 7）

- CI 保持 `openapi:generate + git diff` 漂移检查

### 8) 错误码与错误结构统一（Day 8）

- 统一 `code/message/requestId`
- 前端提示文案与技术错误分层

### 9) 数据与种子脚本健壮性（Day 9-10）

- `db:seed` 幂等与前置检查
- README 增加最短启动路径与故障排查

# 中台基座模板规划（Elysia + Vue3 + Naive UI）

## 1. 项目定位与边界

### 1.1 定位

本项目定位为 **业务中台基座模板**，用于快速搭建各类后台系统的通用骨架。

### 1.2 边界

只实现可复用的基建能力，不内置具体业务模块：

- 做：权限体系、布局体系、CRUD 基建、系统管理、工程化能力
- 不做：文章、评论、订单、支付、营销等业务域模块

### 1.3 设计原则

1. **契约优先**：前后端接口统一规范，类型可追踪
2. **权限优先**：所有页面与动作均可挂接权限点
3. **可配置优先**：通用能力参数化，避免写死
4. **可扩展优先**：业务通过模块注册/路由扩展接入

## 2. 功能分层（仅 P0 / P1）

- 认证与会话（登录、refresh、守卫）
- RBAC 权限体系（用户-角色-权限-菜单）
- 系统管理（用户/角色/菜单）
- CRUD 基建（列表、表单、详情、状态、软删除、上传）
- UI 与工程基础（布局、错误页、请求封装、状态统一）
- P1 增强（字典配置、主题体验、日志检索增强）

## 3. 可执行模块拆分清单

- 后端模块：`auth/account/user/role/menu/permission/dict/config/upload/audit-log/common`
- 前端模块：`modules/auth/permission/system-*` + `components/crud` + `layout` + `infra`
- API 分组：`/api/auth/*`、`/api/users/*`、`/api/roles/*`、`/api/menus/*` 等
- 权限点命名：`{模块}:{资源}:{动作}`

## 4. 里程碑（2-3 个月）

- M1：认证链路 + RBAC 基础 + 基础布局
- M2：CRUD 基建 + 上传/字典/配置 + 错误治理
- M3：体验增强 + 日志检索增强 + 文档与脚手架交付

## 5. 验收标准（Definition of Done）

1. 任意新模块可在 1 天内接入（路由、权限、CRUD 页面）
2. 所有页面动作均可配置权限点并可审计
3. 全局异常、空态、加载表现一致
4. 新项目可在 10-30 分钟内完成基座初始化

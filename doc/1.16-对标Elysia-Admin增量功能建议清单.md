# 1.16 对标 Elysia-Admin 增量功能建议清单（仅未实现项）

## 目标

基于 Harbor 与 `elysia-admin` 的最新代码对比，整理一份**只包含 Harbor 当前未实现**、且在中后台场景中“常见 + 有价值”的功能/页面清单，作为下一阶段迭代输入。

本文明确排除已实现能力（如用户/角色/菜单/部门/字典、在线用户、缓存监控、IP 黑名单、任务中心最小闭环、操作日志）。

## 对比范围

- Harbor：
  - `admin/src/views/`
  - `api/src/modules/`
  - `api/src/app/module-registry.ts`
- Elysia-Admin：
  - `elysia-admin/admin/src/views/`
  - `elysia-admin/server/src/modules/`

## 已实现项（本次不再纳入）

- 系统管理：用户、角色、菜单、部门、字典（主从管理）
- 监控：在线用户、缓存监控、任务中心（最小闭环）
- 安全：IP 黑名单
- 日志：操作日志（审计日志）查询与导出

## 建议增量功能（仅未实现）

## P0（优先做：通用且高频）

### 1) 登录日志页（Login Log）

- 对标来源：
  - `elysia-admin/server/src/modules/system-login-log/`
  - `elysia-admin/admin/src/views/system/log/loginlog/`
- 价值：
  - 与操作日志互补，安全排障时可快速定位“谁在何时何地登录”。
  - 属于中后台常规能力，落地成本可控。
- 建议最小范围：
  - 后端：登录日志查询接口（按账号/IP/时间/结果筛选）。
  - 前端：列表页 + 详情弹窗。
  - 治理：权限点 `system:login-log:view`，接入导出能力可后置。

### 2) API 管理页（接口元数据与权限映射可视化）

- 对标来源：
  - `elysia-admin/server/src/modules/system-api/`
  - `elysia-admin/admin/src/views/system/api/`
- 价值：
  - 让“接口 -> 权限点 -> 菜单/角色”关系可视化，降低权限治理与排障成本。
  - 新成员上手时，能快速理解系统边界。
- 建议最小范围：
  - 后端：聚合当前路由元信息（方法、路径、模块、鉴权、权限码）。
  - 前端：只读列表 + 条件筛选（模块/方法/路径关键字）。
  - 治理：后续再做编辑能力，第一阶段保持只读。

## P1（次优先：平台化增强，已落地）

### 3) 存储管理中心（Storage Management）

- 对标来源：
  - `elysia-admin/server/src/modules/system-storage/`
  - `elysia-admin/admin/src/views/system/storage/`
- 价值：
  - Harbor 已支持 local/oss/cos，缺少“可视化配置与状态管理”入口。
  - 多环境切换、运维交接、配置审计会更清晰。
- 建议最小范围：
  - 后端：读取当前存储配置（脱敏）+ 更新基础配置。
  - 前端：配置页（按存储类型分组）+ 联通性测试按钮。
  - 治理：敏感字段脱敏展示，变更写入审计日志。

**落地状态：已实现（最小版）**

- 前端入口：
  - 菜单路径：`/system/storage`
  - 页面组件：`admin/src/views/StorageManagement.vue`
- 权限点：
  - `system:storage:view`
  - `system:storage:update`
  - `system:storage:test`
- 后端接口：
  - `GET /api/monitor/storage/config`（读取配置，含脱敏视图）
  - `PUT /api/monitor/storage/config`（更新配置到 `sys_configs`）
  - `POST /api/monitor/storage/test`（联通性校验）
- 说明：
  - 支持 `local/oss/cos` 三类配置分组展示。
  - `local` 测试会校验目录可写；`oss/cos` 当前最小版先做配置完整性检查。

### 4) 控制台首页（Dashboard/Console）

- 对标来源：
  - `elysia-admin/admin/src/views/dashboard/console/`
- 价值：
  - 提升“开箱可见价值”，方便演示和运营日常巡检。
  - 可复用现有审计/在线/缓存/任务数据做基础看板。
- 建议最小范围：
  - 前端：轻量看板（今日登录、在线用户数、近期任务执行、操作趋势）。
  - 后端：复用现有统计接口，缺失项按需补聚合接口。

**落地状态：已实现（轻量版）**

- 前端入口：
  - 菜单路径：`/dashboard/console`
  - 页面组件：`admin/src/views/DashboardConsole.vue`
- 权限点：
  - `dashboard:console:view`
- 后端接口：
  - `GET /api/monitor/dashboard/summary`（摘要统计）
  - `GET /api/monitor/dashboard/operation-trend?days=7`（近 N 天趋势）
- 指标范围（当前）：
  - 今日登录数、累计登录数、在线用户数、任务总数、缓存 key 数、操作成功/失败趋势。

## P2（按产品定位选择）

### 5) 用户中心页（User Center）

- 对标来源：
  - `elysia-admin/admin/src/views/system/user-center/`
- 价值：
  - 提供个人资料、头像、密码修改入口，体验更完整。
  - 常见但不影响平台主闭环，可后置。
- 当前处理结果（2026-04-15）：
  - 已落地最小版：个人资料查询/更新、当前账号密码修改、前端页面入口。

### 6) 开放认证流程（注册/忘记密码/重置密码）

> 需要做，我们未来会有web端需求（已按最小闭环落地）

- 对标来源：
  - `elysia-admin/admin/src/views/auth/register/`
  - `elysia-admin/admin/src/views/auth/forget-password/`
  - `elysia-admin/admin/src/views/auth/reset-password/`
- 价值：
  - 对 ToC 或开放平台场景有价值；纯内管系统可不做。
- 建议：
  - 仅在明确业务需要对外账号体系时再纳入。
- 当前处理结果（2026-04-15）：
  - 已落地最小闭环：注册、忘记密码（邮件/短信验证码）、重置密码页面与接口。
  - 说明：验证码优先使用 Redis 存储（无 Redis 时回退内存），通知通过邮件/SMS webhook 通道发送。

### 7) 业务示例模块（订单/商户/支付/退款）

> 我们不需要做，暂时不处理订单相关的

- 对标来源：
  - `elysia-admin/server/src/modules/business-*`
  - `elysia-admin/admin/src/views/business/*`
- 价值：
  - 增强演示性与模板丰富度。
- 建议：
  - 不纳入核心基座默认模块，可做“可选示例包”。
- 当前处理结果（2026-04-15）：
  - 继续维持不纳入基座，保持文档结论不变。

- 必须有权限点与菜单绑定。
- 必须接入审计日志（至少记录写操作）。
- 必须有最小自动化测试（接口集成或页面 smoke 至少一种）。
- 文档需同步：`README.md`、`doc/README.md`、本文件。

# 中台基座数据与接口设计（P0/P1）

> 目标：基于当前模板技术栈（Bun + Elysia + Drizzle + Vue3 + Eden），提供可直接落地的数据库、缓存键、DTO 契约设计。

## 1. 设计约束

1. 仅覆盖 P0/P1 基建能力：认证、RBAC、菜单、用户、字典、配置、上传、审计日志、CRUD 基础能力。
2. 不包含业务域表（订单、内容、评论等）。
3. 全部表统一包含审计字段：`created_at`、`updated_at`、`created_by`、`updated_by`。
4. 软删除场景统一使用：`deleted_at`（为 `null` 表示未删除）。
5. 多租户暂不启用，预留字段可在后续版本补充。

## 2. PostgreSQL 表结构（首批）

## 2.1 用户与权限（RBAC）

### `sys_users`

- 用途：系统用户主表。
- 关键约束：`account` 唯一，`status` 控制启停。

核心字段：

- `id` `bigserial` PK
- `account` `varchar(64)` not null unique
- `password_hash` `varchar(255)` not null
- `nickname` `varchar(64)` not null
- `email` `varchar(128)` null
- `mobile` `varchar(32)` null
- `avatar_url` `varchar(512)` null
- `status` `smallint` not null default `1`（1 启用，0 禁用）
- `last_login_at` `timestamptz` null
- `created_at` `timestamptz` not null default now()
- `updated_at` `timestamptz` not null default now()
- `created_by` `bigint` null
- `updated_by` `bigint` null
- `deleted_at` `timestamptz` null

索引建议：

- `uk_sys_users_account (account)`
- `idx_sys_users_status (status)`
- `idx_sys_users_deleted_at (deleted_at)`

### `sys_roles`

- 用途：角色定义。

核心字段：

- `id` `bigserial` PK
- `code` `varchar(64)` not null unique（如 `admin`、`editor`）
- `name` `varchar(64)` not null
- `description` `varchar(255)` null
- `status` `smallint` not null default `1`
- 审计字段同上

索引建议：

- `uk_sys_roles_code (code)`
- `idx_sys_roles_status (status)`

### `sys_permissions`

- 用途：权限点定义（页面/按钮/API）。

核心字段：

- `id` `bigserial` PK
- `code` `varchar(128)` not null unique（格式：`{模块}:{资源}:{动作}`）
- `name` `varchar(64)` not null
- `type` `smallint` not null（1 菜单权限，2 按钮权限，3 API 权限）
- `module` `varchar(64)` not null
- `description` `varchar(255)` null
- `status` `smallint` not null default `1`
- 审计字段同上

索引建议：

- `uk_sys_permissions_code (code)`
- `idx_sys_permissions_module (module)`

### `sys_user_roles`

- 用途：用户与角色多对多关系。

核心字段：

- `id` `bigserial` PK
- `user_id` `bigint` not null references `sys_users(id)`
- `role_id` `bigint` not null references `sys_roles(id)`
- `created_at` `timestamptz` not null default now()
- `created_by` `bigint` null

约束建议：

- `uk_sys_user_roles_user_role (user_id, role_id)` unique
- `idx_sys_user_roles_user_id (user_id)`
- `idx_sys_user_roles_role_id (role_id)`

### `sys_role_permissions`

- 用途：角色与权限多对多关系。

核心字段：

- `id` `bigserial` PK
- `role_id` `bigint` not null references `sys_roles(id)`
- `permission_id` `bigint` not null references `sys_permissions(id)`
- `created_at` `timestamptz` not null default now()
- `created_by` `bigint` null

约束建议：

- `uk_sys_role_permissions (role_id, permission_id)` unique

## 2.2 菜单与路由

### `sys_menus`

- 用途：动态路由与菜单树。

核心字段：

- `id` `bigserial` PK
- `parent_id` `bigint` not null default `0`
- `name` `varchar(64)` not null（菜单名）
- `route_name` `varchar(128)` not null unique
- `path` `varchar(255)` not null
- `component` `varchar(255)` null（前端组件路径）
- `icon` `varchar(64)` null
- `type` `smallint` not null（1 目录，2 菜单，3 按钮）
- `sort` `int` not null default `0`
- `visible` `smallint` not null default `1`
- `status` `smallint` not null default `1`
- `permission_code` `varchar(128)` null（按钮或页面权限码）
- `keep_alive` `smallint` not null default `0`
- 审计字段同上

索引建议：

- `uk_sys_menus_route_name (route_name)`
- `idx_sys_menus_parent_sort (parent_id, sort)`
- `idx_sys_menus_status_visible (status, visible)`

### `sys_role_menus`

- 用途：角色可见菜单绑定。

核心字段：

- `id` `bigserial` PK
- `role_id` `bigint` not null references `sys_roles(id)`
- `menu_id` `bigint` not null references `sys_menus(id)`
- `created_at` `timestamptz` not null default now()
- `created_by` `bigint` null

约束建议：

- `uk_sys_role_menus (role_id, menu_id)` unique

## 2.3 字典、配置、上传、审计

### `sys_dict_types`

- 用途：字典类型（例如 `user_status`）。

核心字段：

- `id` `bigserial` PK
- `code` `varchar(64)` not null unique
- `name` `varchar(64)` not null
- `status` `smallint` not null default `1`
- `remark` `varchar(255)` null
- 审计字段同上

### `sys_dict_items`

- 用途：字典值。

核心字段：

- `id` `bigserial` PK
- `dict_type_id` `bigint` not null references `sys_dict_types(id)`
- `label` `varchar(64)` not null
- `value` `varchar(64)` not null
- `tag_type` `varchar(32)` null
- `sort` `int` not null default `0`
- `status` `smallint` not null default `1`
- `is_default` `smallint` not null default `0`
- `remark` `varchar(255)` null
- 审计字段同上

约束建议：

- `uk_sys_dict_items_type_value (dict_type_id, value)` unique

### `sys_configs`

- 用途：系统配置中心。

核心字段：

- `id` `bigserial` PK
- `key` `varchar(128)` not null unique
- `value` `text` not null
- `value_type` `smallint` not null default `1`（1 string 2 number 3 bool 4 json）
- `group_name` `varchar(64)` null
- `is_public` `smallint` not null default `0`
- `remark` `varchar(255)` null
- 审计字段同上

### `sys_files`

- 用途：上传文件元数据。

核心字段：

- `id` `bigserial` PK
- `storage` `varchar(32)` not null（local/oss/cos）
- `bucket` `varchar(128)` null
- `object_key` `varchar(255)` not null
- `original_name` `varchar(255)` not null
- `mime_type` `varchar(128)` not null
- `size` `bigint` not null
- `url` `varchar(512)` not null
- `sha256` `varchar(64)` null
- `biz_type` `varchar(64)` null（avatar/editor/image 等）
- `created_at` `timestamptz` not null default now()
- `created_by` `bigint` null

索引建议：

- `idx_sys_files_biz_type (biz_type)`
- `idx_sys_files_created_at (created_at desc)`

### `sys_audit_logs`

- 用途：操作审计日志。

核心字段：

- `id` `bigserial` PK
- `trace_id` `varchar(64)` null
- `operator_user_id` `bigint` null
- `operator_account` `varchar(64)` null
- `action` `varchar(64)` not null（create/update/delete/login/...）
- `module` `varchar(64)` not null
- `resource` `varchar(128)` not null
- `resource_id` `varchar(64)` null
- `request_method` `varchar(16)` not null
- `request_path` `varchar(255)` not null
- `request_ip` `varchar(64)` null
- `request_user_agent` `varchar(512)` null
- `request_payload` `jsonb` null
- `response_code` `int` not null
- `response_message` `varchar(255)` null
- `success` `smallint` not null（1 成功 0 失败）
- `duration_ms` `int` not null default `0`
- `created_at` `timestamptz` not null default now()

索引建议：

- `idx_sys_audit_logs_operator_time (operator_user_id, created_at desc)`
- `idx_sys_audit_logs_module_time (module, created_at desc)`
- `idx_sys_audit_logs_success_time (success, created_at desc)`

## 3. Redis 键设计（首批）

统一前缀：`elysia:admin:`

1. 刷新令牌撤销标记  
   `elysia:admin:auth:refresh:revoked:{jti}`  
   TTL：`refresh_token_expire_seconds`

2. 图形验证码  
   `elysia:admin:auth:captcha:{captcha_id}`  
   TTL：`120s`

3. 登录失败计数  
   `elysia:admin:auth:login:fail:{account}`  
   TTL：`15m`（用于限流/锁定）

4. 用户权限缓存  
   `elysia:admin:perm:user:{user_id}`  
   TTL：`300s`（角色或权限变化时主动删除）

5. 角色菜单缓存  
   `elysia:admin:menu:role:{role_id}`  
   TTL：`300s`

6. 字典缓存  
   `elysia:admin:dict:{dict_code}`  
   TTL：`1800s`

7. 系统配置缓存  
   `elysia:admin:config:{key}`  
   TTL：`1800s`

## 4. API DTO 与响应规范（首批）

## 4.1 统一响应包装

```ts
// 统一响应结构，前后端约定一次即可复用。
export type ApiResponse<T> = {
  code: number; // 0 表示成功，非 0 表示业务失败
  message: string;
  data: T;
  requestId?: string;
};

// 分页结构，列表接口统一返回。
export type PageData<T> = {
  list: T[];
  total: number;
  page: number;
  pageSize: number;
};
```

错误码建议：

- `0` 成功
- `40001` 参数错误
- `40101` 未登录或 token 无效
- `40301` 无权限
- `40401` 资源不存在
- `40901` 资源冲突（唯一键冲突）
- `50001` 服务内部异常

## 4.2 认证模块 DTO

### `POST /api/auth/login`

请求：

```json
{
  "account": "admin",
  "password": "admin123",
  "captchaId": "cpt_xxx",
  "captchaCode": "8A7K",
  "rememberMe": true
}
```

响应 `data`：

```json
{
  "accessToken": "xxx",
  "refreshToken": "xxx",
  "expiresIn": 3600,
  "user": {
    "id": 1,
    "account": "admin",
    "nickname": "管理员",
    "roles": ["admin"],
    "permissions": ["system:user:view"]
  }
}
```

### `POST /api/auth/refresh`

请求：

```json
{
  "refreshToken": "xxx"
}
```

响应 `data`：

```json
{
  "accessToken": "new_xxx",
  "refreshToken": "new_xxx",
  "expiresIn": 3600
}
```

### `POST /api/auth/logout`

请求：

```json
{
  "refreshToken": "xxx"
}
```

响应：`data = null`

## 5. 落地顺序（建议）

1. 建表顺序：`sys_users -> sys_roles -> sys_permissions -> 关系表 -> sys_menus -> dict/config/file/audit`
2. 初始化种子：角色（admin/editor）、管理员账号、菜单与权限点
3. 先实现认证和权限查询接口，再实现用户/角色/菜单管理
4. 前端先接入登录、动态菜单、按钮权限，再接用户与角色页面
5. 关键缓存（权限、菜单、配置、字典）在更新接口中主动失效

## 6. 与当前仓库对齐说明

1. 当前 `api/src/infra/db/schema.ts` 仅有 `users/articles` 示例表，建议按本方案扩展为 `sys_*` 命名体系。
2. 当前鉴权已具备 refresh token 撤销能力（Redis + 内存兜底），可直接复用本方案 Redis key 规范。
3. 当前 `admin/web` 已使用 Eden 类型联动，DTO 按本方案定义后可直接透传为前端类型来源。

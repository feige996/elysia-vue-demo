# 1.15 P0 功能搬运执行计划

## 目标

基于 `1.14-对标Elysia-Admin差异与功能搬运清单`，将 Harbor 的 P0 搬运项落为可执行任务，确保：

- 任务可拆分到具体文件与接口。
- 每项有最小验收与测试口径。
- 继续遵循 Harbor 的“默认最小集 + feature flag 按需启用”原则。

本期范围仅包含 P0：

1. 监控任务中心（任务列表 + 启停 + Cron 管理）
2. 部门管理与数据权限基础能力
3. 字典管理后台化（补齐写操作）

## 迭代节奏（建议 2 周）

- 第 1 周：后端能力优先（接口、数据模型、权限点、审计日志）
- 第 2 周：管理端页面 + 联调 + 自动化测试 + 文档收口

## 共性约束（所有 P0 模块必须满足）

- 所有新增模块均通过 `api/src/shared/config/env.ts` 增加开关，并接入 `api/src/app/module-registry.ts`。
- 默认关闭：不开启时路由不挂载、`/ready` 返回 disabled 状态。
- 新增管理接口必须挂 RBAC 权限点并记录审计日志。
- 至少补齐 1 类自动化测试（单元或集成），建议关键路径做集成测试。

## A. 监控任务中心（P0-1）

## A1 后端任务

- 新增模块：`api/src/modules/job/`（建议结构）
  - `job.module.ts`：路由定义
  - `job.service.ts`：任务启停、调度、执行触发
  - `job.repository.ts`：任务配置读写
  - `dto/job.dto.ts`：请求/响应 schema
- 新增数据表（或扩展现有 schema）
  - `sys_jobs`：`id/name/cron/status/args/lastRunAt/nextRunAt/remark/createdAt/updatedAt`
- 新增 API（最小闭环）
  - `GET /api/jobs`：分页查询
  - `POST /api/jobs`：新增任务
  - `PUT /api/jobs/:id`：编辑任务（含 cron）
  - `POST /api/jobs/:id/toggle`：启停任务
  - `POST /api/jobs/:id/run`：手动触发一次
- 队列与调度
  - 若 `features.queue` / `features.cron` 关闭：返回明确错误（功能未启用）
  - 若开启：注册到队列调度器并更新下一次执行时间

## A2 前端任务（admin）

- 新增页面：`admin/src/views/JobManagement.vue`
- 页面功能
  - 列表查询（名称、状态、cron）
  - 新增/编辑弹窗
  - 启停开关
  - 立即执行按钮
  - 最近执行信息展示（成功/失败、耗时）
- 权限点
  - `job:list` `job:create` `job:update` `job:toggle` `job:run`

## A3 测试与验收

- API 集成测试
  - 开关关闭时访问返回 404 或 feature-disabled（按现有约定）
  - 开关开启后可完成新增 -> 启用 -> 触发 -> 查询链路
- 管理端最小 E2E
  - 登录后进入任务页，新增任务并看到列表更新
- 验收标准
  - 任务启停状态可持久化
  - cron 配置变更后可生效
  - 审计日志可检索到关键操作

## B. 部门管理与数据权限基础能力（P0-2）

## B1 后端任务

- 新增模块：`api/src/modules/dept/`
  - 树结构查询 + 增删改
  - 状态启停
- 新增表：`sys_depts`（最小字段）
  - `id/parentId/name/code/sort/status/leader/phone/email/createdAt/updatedAt`
- API（最小闭环）
  - `GET /api/depts/tree`
  - `POST /api/depts`
  - `PUT /api/depts/:id`
  - `POST /api/depts/:id/toggle`
  - `DELETE /api/depts/:id`（有子节点时阻止删除）
- 数据权限最小落地
  - 用户模型增加 `deptId`
  - 用户列表支持按 `deptId` 过滤
  - 预留角色数据范围字段（先不做复杂策略引擎）

## B2 前端任务（admin）

- 新增页面：`admin/src/views/DeptManagement.vue`
- 页面功能
  - 左侧部门树 + 右侧详情/编辑
  - 新增子部门、编辑、启停、删除
  - 用户页联动部门筛选
- 权限点
  - `dept:list` `dept:create` `dept:update` `dept:delete` `dept:toggle`

## B3 测试与验收

- API 集成测试
  - 树结构查询正确
  - 禁止删除含子节点部门
  - 用户按部门过滤正确
- 验收标准
  - 部门树可稳定维护
  - 用户管理可按部门筛选
  - 审计日志包含部门变更记录

## C. 字典管理后台化（P0-3）

## C1 后端任务

- 在现有 `api/src/modules/system/system.module.ts` 基础上扩展写接口（避免新模块分散）
- 补齐 API
  - 字典类型：`GET/POST/PUT/DELETE /api/dict-types`
  - 字典项：`GET/POST/PUT/DELETE /api/dict-items`
  - 状态切换：`POST /api/dict-types/:id/toggle`、`POST /api/dict-items/:id/toggle`
- 约束校验
  - 字典类型编码唯一
  - 删除字典类型前必须先清空字典项（或阻止删除）
  - 字典项排序字段合法性校验

## C2 前端任务（admin）

- 新增页面：`admin/src/views/DictTypeManagement.vue`
- 新增页面：`admin/src/views/DictItemManagement.vue`（或主从一体页）
- 页面功能
  - 字典类型列表、搜索、启停、增删改
  - 类型下字典项管理（标签、值、排序、状态、默认）
  - 与业务页联动验证（例如用户状态/标签下拉读取）
- 权限点
  - `dict:type:list/create/update/delete/toggle`
  - `dict:item:list/create/update/delete/toggle`

## C3 测试与验收

- API 集成测试
  - 新增类型 + 新增项 + 查询项链路通过
  - 删除保护与唯一约束生效
- 前端联调验证
  - 下拉字典项实时反映后台修改
- 验收标准
  - 字典不再依赖手工 SQL 维护
  - 变更可审计、可回溯

## D. 任务拆分清单（可直接建 Issue）

- `P0-1-BE` 任务中心后端模块与表结构
- `P0-1-FE` 任务中心页面与交互
- `P0-1-QA` 任务中心集成测试与 E2E smoke
- `P0-2-BE` 部门管理与用户部门过滤
- `P0-2-FE` 部门管理页 + 用户页联动
- `P0-2-QA` 部门树与删除保护测试
- `P0-3-BE` 字典类型/字典项写接口与约束
- `P0-3-FE` 字典后台管理页
- `P0-3-QA` 字典联调与回归测试
- `P0-DOC` README 与 doc 文档同步更新

## E. 交付检查清单（发布前）

- `bun run --cwd api typecheck` 通过
- `bun run --cwd api test` 通过
- `bun run --cwd admin typecheck` 通过
- 根目录 `bun run dev:check` 通过
- 文档更新：`1.01`、`1.12`、`1.14`、本文件

## F. 风险与应对

- 队列/调度依赖引入后本地环境复杂度上升
  - 应对：默认关闭，提供最小本地启动说明
- 部门数据与现有用户/角色关联改造可能影响旧数据
  - 应对：迁移脚本提供默认部门兜底
- 字典后台化后写操作权限边界不清
  - 应对：先收敛到管理员角色，后续再做细粒度授权

## G. 完成定义（DoD）

- P0 三项能力均具备“接口 + 页面 + 权限 + 审计 + 测试 + 文档”六件套。
- 模块可通过 feature flag 独立启停。
- 核心演示路径可在 15 分钟内完整走通（登录 -> 配置/管理 -> 查询验证）。

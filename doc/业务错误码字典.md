# Harbor 业务错误码字典

> 目标：统一后端错误码语义与前端提示策略，避免页面各自硬编码判断。

## 通用约定

- 成功：`code = 0`
- 失败：`code != 0`
- 响应结构：
  - `code`: 业务错误码
  - `message`: 后端默认消息
  - `requestId`: 请求追踪 ID
  - `data`: 成功场景的业务数据

## 错误码定义

| code   | key                 | HTTP status | 语义               | 前端默认提示                   |
| ------ | ------------------- | ----------- | ------------------ | ------------------------------ |
| 400000 | BAD_REQUEST         | 400         | 请求不合法         | 请求参数有误，请检查后重试     |
| 400100 | VALIDATION_ERROR    | 400         | 参数校验失败       | 提交数据校验失败，请检查输入项 |
| 401000 | UNAUTHORIZED        | 401         | 未登录或登录态失效 | 登录状态已失效，请重新登录     |
| 401001 | INVALID_CREDENTIALS | 401         | 账号密码错误       | 账号或密码错误                 |
| 403000 | FORBIDDEN           | 403         | 无权限访问目标资源 | 当前账号无权限执行该操作       |
| 404000 | NOT_FOUND           | 404         | 资源不存在         | 请求的资源不存在或已被删除     |
| 409000 | CONFLICT            | 409         | 资源冲突           | 数据冲突，请刷新后重试         |
| 429000 | RATE_LIMITED        | 429         | 触发限流           | 请求过于频繁，请稍后再试       |
| 500000 | INTERNAL_ERROR      | 500         | 服务器内部错误     | 服务暂时不可用，请稍后再试     |
| 503000 | SERVICE_UNAVAILABLE | 503         | 服务不可用         | 服务暂不可用，请稍后再试       |

## 前端落地约定

- 统一通过 `admin/src/api/error-map.ts` 做错误码 -> 提示文案映射。
- 页面层不再手写 `if (error.code === ...)`，统一调用映射工具。
- 页面可以传入业务场景兜底文案（例如“加载失败”“保存失败”），未命中映射时使用兜底。

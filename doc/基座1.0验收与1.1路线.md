# Harbor 基座 1.0 验收与 1.1 路线

> 目标：明确“当前是否达到可复用基座标准”，并给出可执行的下一阶段演进计划。

> 当前状态口径：`已完成` = 已有代码/配置落地；`待验证` = 需要再次执行命令或在 CI 复核；`未开始` = 尚未实施。

## 一、1.0 验收清单（Release Gate）

### 1) 核心能力

- [x] RBAC 关键链路可用（登录、菜单、权限、角色、用户管理）- 状态：已完成
- [x] 审计日志可查询（含操作者、时间、结果、模块维度）- 状态：已完成
- [x] 统一错误码映射已接入主要后台页面 - 状态：已完成
- [x] `/ready` 结构化输出可定位依赖状态（database/redis）- 状态：已完成
- [x] 分级限流与 CORS 白名单配置可用 - 状态：已完成

### 2) 质量与交付

- [ ] 本地通过：`bun run dev:check` - 状态：待验证
- [ ] 本地通过：`bun run test` - 状态：待验证
- [ ] 本地通过：`bun run build` - 状态：待验证
- [ ] admin e2e smoke 在本地和 CI 稳定通过 - 状态：待验证
- [x] 关键文档可交接（README、CONTRIBUTING、执行清单）- 状态：已完成

### 3) 复用与规范

- [x] CRUD 组件基建已用于核心页面（SearchBar / TableToolbar / FormDrawer / UnifiedState）- 状态：已完成（已沉淀 `DataTablePage` 并迁移核心列表页）
- [x] 代码结构与提交规范在团队内统一（无“页面内散装请求/散装错误处理”）- 状态：基本完成
- [ ] 已知安全风险有台账、优先级和处理计划 - 状态：待完善

## 二、1.1 路线图（按优先级）

### P0（安全与稳定性）

1. 依赖安全专项清理（high/critical）
2. 数据库层升级回归（尤其 ORM 与迁移链路）
3. e2e 稳定性提升（提升可复跑性与失败诊断信息）

### P1（可观测与运维）

4. 审计日志增强：导出、聚合统计、关键操作告警
5. `/ready` + 指标体系接入（Prometheus/Grafana 或等效）
6. 发布与回滚流程脚本化（减少人工步骤）

### P2（复用与效率）

7. CRUD 组件 2.0：沉淀 `DataTablePage` 模板（当前状态：已完成）
8. 模块脚手架增强：一键生成 API + 页面 + 路由 + 权限点
9. 前端构建体积持续优化（UI 库按需策略与更细分包）
10. 版本化治理：`v1.x` 变更日志与升级指南

---

## 三、四周执行计划（可直接排期）

### 第 1 周：安全与质量基线

- [ ] 完成依赖风险分级与处理决策（修复/暂缓/替代）- 当前优先级：高
- [ ] 升级可控依赖并回归 `dev:check + test + build` - 当前优先级：高
- [ ] 补齐安全台账文档（风险、影响、处理时间）- 当前优先级：高

### 第 2 周：运维可观测

- [ ] 审计日志导出与聚合统计最小版
- [ ] 接入基础运行指标与错误告警
- [ ] 优化 `/ready` 与故障定位手册

### 第 3 周：复用提效

- [ ] 落地 `DataTablePage` 模板并迁移至少 1 个页面
- [ ] 输出页面开发约定（查询、弹窗、状态、错误统一规范）
- [ ] 清理重复代码，降低新页面开发成本

### 第 4 周：发布与版本治理

- [ ] 发布/回滚脚本化（含数据库迁移步骤）
- [ ] 输出 `v1.1.0` 发布说明与升级指南
- [ ] 完成一次演练：发布 -> 验证 -> 回滚（桌面演练或预发演练）

---

## 四、完成判定（Definition of Done）

- [ ] 关键风险项有闭环（处理或明确暂缓与复评时间）
- [ ] 交付流程稳定（CI + 本地回归一致）
- [ ] 新增页面能按基建模板快速交付
- [ ] 文档可指导新人 1 天内完成本地运行与开发
- [ ] 版本升级有记录、可追溯、可回滚

---

## 五、本周建议执行（从现在开始）

1. 执行并记录：`bun run dev:check`、`bun run test`、`bun run build`、admin e2e smoke。
2. 将第 2 节中的 4 个“待验证”项按结果改为 `[x]` 或补充阻塞原因。
3. 建立“依赖风险台账”最小版（风险等级、影响范围、处理策略、截止时间、负责人）。
4. 形成第一版发布/回滚脚本清单（哪怕半自动），并写入 `README` 或独立运维文档。

# 1.10 当前状态与验收口径

## 目标

统一团队对“当前完成到什么程度、还差什么”的判断口径，避免口头对齐偏差。

## 状态定义

- `已完成`：代码与配置已落地，关键链路可用。
- `待验证`：需执行命令或 CI 再次确认。
- `待完善`：已具备基础能力，但仍缺闭环动作。
- `未开始`：尚未落地实现。

## 当前状态（基于最近一次回归）

### 核心能力

- RBAC（用户/角色/菜单/权限）：`已完成`
- 审计日志查询：`已完成`
- 统一错误码映射：`已完成`
- `/ready` 依赖状态输出：`已完成`
- 分级限流与 CORS：`已完成`

### 质量与交付

- `bun run dev:check`：`待验证`
- `bun run test`：`待验证`
- `bun run build`：`待验证`
- `bun run test:e2e:admin`：`待验证`

### 治理与风险

- 依赖风险台账：`已建立，待持续更新`
- 发布/回滚脚本化：`待完善`
- 指标告警体系：`未开始`
- 可选模块开关框架（默认最小集）：`未开始`

## 验收口径（Release Gate）

进入“可发布”状态需满足：

1. 四项命令全部通过：`dev:check`、`test`、`build`、`test:e2e:admin`。
2. 无阻塞级别已知风险（或有明确暂缓记录与复评时间）。
3. 文档入口可用（README + doc 索引 + 关键规范文档）。

## 维护规则

- 每次大改后必须更新本文档状态。
- 每周例会前至少更新一次“待验证/待完善”项。
- 与 `1.12-路线图与Backlog` 联动维护，避免重复记录。

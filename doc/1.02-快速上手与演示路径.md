# 1.02 快速上手与演示路径

## 10 分钟体验路径

### 1) 启动项目

在仓库根目录执行：

```bash
bun install
bun run --cwd api db:migrate
bun run --cwd api db:seed
bun run dev
```

默认访问地址：

- API：`http://localhost:9000`
- Admin：`http://localhost:7000`
- Web：`http://localhost:8000`

### 2) 登录后台

本地示例账号：

- `admin` / `admin123`
- `editor` / `editor123`

### 3) 体验核心能力

建议按以下顺序演示：

1. 登录 -> 查看菜单与权限控制（`admin` 与 `editor` 对比）。
2. 进入用户管理：新增用户、编辑用户、批量删除。
3. 进入角色管理：创建角色、分配权限、分配菜单。
4. 查看审计日志：按操作者/模块/结果筛选。

### 4) 验证工程质量能力

```bash
bun run dev:check
bun run test
bun run build
bun run test:e2e:admin
```

## 常见问题

- 登录后权限异常：确认已执行 `db:seed`，并重新登录刷新权限上下文。
- `/ready` 显示依赖未就绪：检查 PostgreSQL/Redis 连通性与环境变量。
- E2E 失败：确认 API 地址可访问，且数据库已迁移并完成种子初始化。

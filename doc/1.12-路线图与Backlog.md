# 1.12 路线图与 Backlog

> 仅保留“未完成或待完善”事项，避免与历史完成项混写。

## P0（优先处理）

1. 依赖安全专项清理（high/critical）
2. ORM 与迁移链路升级回归
3. E2E 稳定性与失败诊断信息增强
4. 监控任务中心最小闭环（任务 CRUD / 启停 / 手动触发）

## P1（建议尽快推进）

1. 审计日志导出与聚合统计
2. 指标与告警体系接入（如 Prometheus/Grafana）
3. 发布/回滚脚本化（减少人工步骤）

## P2（提效与演进）

1. 模块脚手架增强（API + 页面 + 路由 + 权限点）
2. 前端构建体积持续优化
3. 版本化治理（changelog 与升级指南）
4. 可选模块开关落地（默认最小集，增强能力按配置启用）

## P0 搬运进展（对标 Elysia-Admin）

- 已完成：字典管理后台化（类型/字典项 CRUD + 状态切换）。
- 已完成：部门管理最小闭环（树结构 + 增删改 + 状态切换 + 删除保护）。
- 已完成：用户按部门筛选与所属部门维护。
- 待完成：任务中心最小闭环（`monitor-job` 方向）。

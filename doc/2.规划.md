# 中台基座模板规划（Elysia + Vue3 + Naive UI）

## 1. 项目定位与边界

### 1.1 定位

本项目定位为 **业务中台基座模板**，用于快速搭建各类后台系统的通用骨架。

### 1.2 边界

只实现可复用的基建能力，不内置具体业务模块：

- 做：权限体系、布局体系、CRUD 基建、系统管理、工程化能力
- 不做：文章、评论、订单、支付、营销等业务域模块

### 1.3 设计原则

1. **契约优先**：前后端接口统一规范，类型可追踪
2. **权限优先**：所有页面与动作均可挂接权限点
3. **可配置优先**：通用能力参数化，避免写死
4. **可扩展优先**：业务通过模块注册/路由扩展接入

## 2. 功能分层（仅 P0 / P1）

## 2.1 P0（必须交付）

### A. 认证与会话

1. 账号密码登录
2. 图形验证码
3. 记住我、自动登录
4. JWT 鉴权（Access Token + Refresh Token）
5. Token 刷新与过期兜底
6. 路由守卫（未登录拦截、无权限拦截）
7. 自动登出（多端失效/主动失效）

### B. 权限体系

1. RBAC（用户-角色-权限）
2. 菜单权限（按角色返回动态菜单）
3. 按钮权限（新增/编辑/删除/导入导出等动作控制）
4. 权限指令（前端统一 `v-permission`）
5. 操作日志（谁、何时、对什么做了什么）

### C. 系统管理

1. 用户管理：列表、搜索、新增、编辑、禁用、重置密码
2. 角色管理：权限勾选、角色状态
3. 菜单管理：多级菜单、路由绑定、图标配置、排序

### D. CRUD 基建

1. 列表页：筛选、分页、排序、批量操作
2. 表单页：新增、编辑、校验
3. 详情页：标准只读详情
4. 状态切换：启用/禁用
5. 回收站/软删除：删除、恢复、彻底删除
6. 文件上传：单图、多图、富文本图片（统一上传服务）

### E. UI 与工程基础

1. 布局：侧边栏 + 顶部导航 + 多标签页（tabs）
2. 401/404/500 错误页
3. 请求封装：鉴权头、错误码、重试与提示
4. loading / 异常 / 空状态统一处理

## 2.2 P1（增强交付）

### A. 配置与字典

1. 字典管理：状态、分类、开关等全局常量
2. 系统配置中心：动态读取与缓存

### B. 体验增强

1. 暗黑模式
2. 主题色切换
3. 菜单折叠
4. 全屏
5. 面包屑
6. 返回顶部

### C. 通用能力增强

1. 排序拖拽（列表通用能力）
2. 复制文本、代码高亮
3. 操作日志检索增强（按用户/模块/时间过滤）

## 3. 可执行模块拆分清单

## 3.1 后端模块（Bun + Elysia）

1. `auth`：登录、登出、refresh、验证码、会话检查
2. `account`：当前用户信息、个人资料、密码修改
3. `user`：用户 CRUD、启停、重置密码、角色绑定
4. `role`：角色 CRUD、角色权限绑定
5. `menu`：菜单树、菜单 CRUD、角色菜单关联
6. `permission`：权限点定义、角色权限关联、权限校验
7. `dict`：字典项 CRUD、缓存与下发
8. `config`：系统配置 CRUD、配置缓存
9. `upload`：文件上传、文件元数据管理
10. `audit-log`：操作日志写入、查询、筛选
11. `common`：分页、统一响应、错误码、参数校验、中间件

## 3.2 前端模块（Vue3 + TypeScript + Naive UI）

1. `modules/auth`：登录页、验证码、token 管理、自动登录
2. `modules/permission`：路由守卫、权限指令、按钮权限组件
3. `modules/system-user`：用户管理页（列表/表单/详情）
4. `modules/system-role`：角色管理页（权限树勾选）
5. `modules/system-menu`：菜单管理页（树形结构、排序）
6. `modules/dict-config`：字典与系统配置页面
7. `modules/upload`：上传组件封装与文件选择器
8. `modules/audit-log`：操作日志检索与详情
9. `components/crud`：通用 `SearchBar`、`DataTable`、`FormDrawer`
10. `layout`：侧边栏、顶栏、tabs、面包屑、主题切换
11. `infra`：request 封装、错误处理、全局状态、缓存策略

## 3.3 API 契约分组（建议）

1. `/api/auth/*`：`login`、`refresh`、`logout`、`captcha`
2. `/api/users/*`：用户管理相关
3. `/api/roles/*`：角色管理与权限绑定
4. `/api/menus/*`：菜单树与菜单维护
5. `/api/permissions/*`：权限点查询与分配
6. `/api/dicts/*`：字典管理
7. `/api/configs/*`：系统配置管理
8. `/api/uploads/*`：上传与文件记录
9. `/api/audit-logs/*`：日志查询

## 3.4 权限点命名规范（建议）

统一格式：`{模块}:{资源}:{动作}`

示例：

1. `system:user:view`
2. `system:user:create`
3. `system:user:update`
4. `system:user:delete`
5. `system:role:assign-permission`
6. `system:menu:sort`
7. `system:config:update`
8. `audit:log:view`

## 4. 里程碑（2-3 个月）

### M1（第 1 个月）

1. 完成认证链路与 RBAC 基础闭环
2. 打通用户/角色/菜单管理
3. 完成基础布局与路由守卫

### M2（第 2 个月）

1. 完成 CRUD 基建组件（列表/表单/详情）
2. 完成上传、字典、系统配置
3. 完成错误页与请求层统一治理

### M3（第 3 个月）

1. 完成暗黑模式、主题、全屏等体验增强
2. 完成日志检索增强与拖拽排序能力
3. 完成文档、脚手架初始化模板、交付验收

---

## 5. 验收标准（Definition of Done）

1. 任意新模块可在 1 天内接入（路由、权限、CRUD 页面）
2. 所有页面动作均可配置权限点并可审计
3. 全局异常、空态、加载表现一致
4. 新项目可在 10-30 分钟内完成基座初始化

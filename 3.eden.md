# Elysia 最强黑科技：**后端路由 schema 自动推全量接口类型，前端直接 import，不用手写任何 interface**

## 一句话原理

- 后端用 `t.xxx` 写接口入参/出参校验
- Elysia **自动推导全路由请求、参数、返回值 TS 类型**
- 直接把整个 app 导出成 `type App`
- admin、web 前端**直接 import 这个 App 类型**
- 搭配官方 **Eden / Eden Fetch**，调用接口**零类型手写、全链路提示、0 不一致**

## 你三端结构直接这么玩（完美适配）

### 1、后端 server（只写一遍，源头唯一）

```ts
// server/src/index.ts
import { Elysia, t } from 'elysia';

const app = new Elysia().get(
    '/user/:id',
    ({ params }) => {
        return { id: params.id, name: 'test' };
    },
    {
        params: t.Object({ id: t.String() }), // Elysia自动抽类型
    },
);

export type App = typeof app; // 导出全接口类型！！
```

### 2、admin 后台 Vue、web 前台 Vue 直接用

安装客户端：

```bash
bun add @elysiajs/eden
```

前端直接引用后端类型：

```ts
// admin/src/api/eden.ts
import { treaty } from '@elysiajs/eden';
// 直接导入后端App全量类型！！
import type { App } from '../../server/src/index';

// 全路由、入参、返回值100%自动补齐
const api = treaty<App>('http://localhost:3000');

// 调用自动提示，返回值全自动类型
const res = await api.user[':id'].get({ params: { id: '1' } });
```

## 重点：为什么别人说「前端不能直接用」？

很多人误区：

- ❌ 不能**运行时导入后端 js 代码**（node环境不能跑浏览器代码）
- ✅ 可以**纯 import type 只拿TS类型**，完全不打包、不进浏览器运行代码

TS 编译会直接丢掉类型，**零体积、零运行成本**
Bun + Vite 完美兼容，毫无问题
